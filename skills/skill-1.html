Contexte du projet

Ce projet consiste en la réalisation d'un émulateur permettant d'exécuter des lignes d'un langage d'assemblage (ASM) personnalisé, conçu pour une architecture de processeur créée sur mesure.

Lien avec l'AC21.03 : Adopter de bonnes pratiques

La validation de cette compétence s'appuie sur la rigueur architecturale et méthodologique appliquée durant le développement :

  • Architecture : Mise en place de l'architecture d'exécution des instructions (basée sur le Design Pattern Commande), d'une architecture d'interruptions et d'interactions avec des ports d'entrée/sortie.
  • Méthodologie TDD : Le développement a été piloté par les tests (Test Driven Development). Les tests unitaires ont été rédigés en amont avec Google Test en respectant le standard AAA (Arrange, Act, Assert) avant même l'implémentation des méthodes.

Limites & Solutions envisagées

Lors de l'analyse des performances de l'architecture logicielle retenue, j'ai identifié le point d'optimisation suivant :

Limite : Le fait d'utiliser le design pattern commande amène à devoir manipuler une liste de pointeurs, ce qui n'est pas optimal en termes de performances car le cache CPU n'est pas exploité à son plein potentiel (cache miss fréquents).

Solution proposée : Utiliser une implémentation permettant d'avoir une liste de branches pour exécuter chaque instruction (ex: grand switch/case ou tables de sauts), plutôt que d'instancier une classe fille par instruction ASM.
