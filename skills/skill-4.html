Contexte du projet

Ce projet visait à développer un ERP complet à destination des professeurs de l'IUT. Le système repose sur une architecture distribuée composée de 3 applications distinctes (un Frontend, un Backend, et une Base de données) qui doivent communiquer entre elles de manière fluide et sécurisée.

Lien avec l'AC23.01 : Développer des applications communicantes

J'ai eu la charge de concevoir la couche intermédiaire (middleware) du projet :

  • Exposition de l'API : Mise en place des routes REST sur le backend permettant au frontend d'effectuer des requêtes HTTP (appels clients).
  • Liaison de données (ORM) : Configuration de la communication entre le backend (Spring Boot) et la base de données en utilisant le framework Hibernate. Le backend joue ainsi un rôle d'interface indispensable pour que le frontend puisse manipuler les données.

Difficulté technique & Solution

L'infrastructure réseau imposait des contraintes fortes sur la communication :

Difficulté : La base de données était hébergée sur un serveur distant, externe à notre environnement de développement local. Établir une connexion sécurisée s'est avéré complexe, car il fallait absolument éviter d'exposer et d'ouvrir le port de la base de données directement sur le réseau internet public.

Solution : Pour contourner ce problème, j'ai mis en place un tunnel chiffré. À chaque connexion du backend vers la base de données, les communications transitent de manière sécurisée via une connexion SSH, garantissant l'étanchéité du réseau.
